# Getting Started with Flux

This guide walks you from installation to running your first backtest.

## Prerequisites

You need a working **Rust toolchain** with `cargo` available. If you don't have Rust installed, visit [rustup.rs](https://rustup.rs) and follow the instructions for your platform.

Verify your installation:

```bash
cargo --version
```

## Building from Source

Clone the repository and build in release mode:

```bash
git clone https://github.com/thefuzzybear/flux-lang.git
cd flux-lang
cargo build --release
```

The `flux` binary will be available at `target/release/flux-cli`. You can add it to your `PATH` or invoke it directly via `cargo run -p flux-cli --`.

## Project Initialization

Scaffold a new Flux project with `flux init`:

```bash
flux init my-strategy
cd my-strategy
```

This creates the following structure:

```
my-strategy/
├── flux.toml              # Project manifest
├── strategies/
│   └── example.flux       # Example SMA crossover strategy
├── data/
│   └── sample.csv         # Sample OHLCV data
├── README.md
└── .gitignore
```

- **flux.toml** — Project configuration (name, version, directory paths)
- **strategies/** — Your `.flux` strategy files live here
- **data/** — CSV market data for backtesting

## Writing a Strategy

A Flux strategy has three sections: `params` for configurable constants, `state` for variables that persist across bars, and `on bar` for the logic that runs on each price bar.

Here's a simple moving average crossover strategy:

```flux
from indicators import {sma}

strategy SmaCrossover {
    params {
        period = 5
    }

    state {
        bar_count = 0
    }

    on bar {
        bar_count = bar_count + 1
        avg = sma(close, period)

        if close > avg and not in_position {
            OPEN(symbol, 100.0)
        }

        if close < avg and in_position {
            CLOSE(symbol)
        }
    }
}
```

Key concepts:

- **`from indicators import {sma}`** — Import indicator functions before using them
- **`params`** — Constants you can tune without changing logic
- **`state`** — Variables that carry values from one bar to the next
- **`on bar`** — Runs once per data row; has access to `close`, `open`, `high`, `low`, `volume`, `symbol`, and `in_position`
- **`sma(value, period)`** — Built-in simple moving average indicator
- **`OPEN(symbol, qty)`** — Open a position with the given quantity
- **`CLOSE(symbol)`** — Close the entire position for a symbol

## Sample CSV Data

Flux backtests require OHLCV data in CSV format with these columns: `timestamp`, `symbol`, `open`, `high`, `low`, `close`, `volume`.

Save the following as `data/sample.csv`:

```csv
timestamp,symbol,open,high,low,close,volume
2024-01-02,AAPL,185.50,186.75,185.10,186.20,1200000
2024-01-03,AAPL,186.20,186.90,185.80,186.50,980000
2024-01-04,AAPL,186.50,187.10,186.30,186.80,750000
2024-01-05,AAPL,186.80,187.25,186.60,187.00,620000
2024-01-08,AAPL,187.00,187.50,186.90,187.30,830000
2024-01-09,AAPL,187.30,187.60,186.80,186.90,540000
2024-01-10,AAPL,186.90,187.20,186.50,186.00,670000
```

Column names are case-insensitive and can appear in any order. Each row represents one bar for one symbol.

## Running `flux check`

Before backtesting, validate your strategy for syntax and type errors:

```bash
flux check strategies/example.flux
```

If there are no errors, the command exits silently with code 0. If there are problems, you'll see error messages with source locations:

```
error: strategies/example.flux:12:9 — unknown variable 'closee'
```

## Running `flux backtest`

Run your strategy against market data:

```bash
flux backtest strategies/example.flux --data data/sample.csv --capital 10000
```

- **`--data`** — Path to the CSV file with price data
- **`--capital`** — Starting capital for the simulation (default: 10000)

## Reading the Output

The backtest produces three sections:

### Signals

Raw trading signals generated by your strategy on each bar:

```
--- Signals ---
  0 Open AAPL 100
  5 Close AAPL
```

Each line shows the bar index, signal type (`Open`, `Close`, or `CloseQty`), symbol, and quantity.

### Fills

Simulated executions of your signals:

```
--- Fills ---
  Bar    0 |  BUY | AAPL     100.00 @     186.20
  Bar    5 | SELL | AAPL     100.00 @     186.90
```

Fills show when positions were actually opened (`BUY`) or closed (`SELL`), with the fill price and quantity.

### Portfolio Summary

End-of-backtest portfolio metrics:

```
--- Portfolio Summary ---
  Initial Capital:       10000.00
  Final Equity:          10070.00
  Realized P&L:             70.00
  Unrealized P&L:            0.00
  Total Return:            0.70%
  Open Positions:              0
  Gross Exposure:            0.00
  Net Exposure:              0.00
  Total Fills:                 2
```

This tells you the final equity, realized and unrealized profit/loss, return percentage, and exposure metrics.
